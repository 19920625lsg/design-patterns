# UML类图详解

## 例图详解
> ![类图例子](https://img.mukewang.com/szimg/5c1d9c4600011c4819201080.jpg)
+ 最上面的长方形：类的名字
  + 普通类 `标准字体`
  + 抽象类 `斜体`，图中标识错误，因为存在抽象方法study()，所以是抽象类，应该用斜体;
  + 接口 `<interface>`
  
+ 中间的长方形：`类的属性`
  + `加号 +`: 最大的权限，public
  + `减号 -`：最小的权限，private
  + `井号 #`#protected
~default
——横线代表static属性

斜体代表抽象方法
